# خريطة الارتباطات الكاملة

## 1) نقطة الدخول وسلسلة التحميل

- `src/main.tsx` -> `src/app/App.tsx` -> `src/app/routes.ts`.
- `src/app/App.tsx` يركّب المزودات بالترتيب:
  - `AuthProvider`
  - `ProductProvider`
  - `CartProvider`
- `src/app/components/Layout.tsx` يغلف الصفحات العامة ويحقن:
  - `Header`
  - `Outlet`
  - `Footer`

## 2) الارتباطات الدقيقة: Routes -> Pages -> Components -> Data

### `/`
- Page: `src/app/pages/Home.tsx`
- يعتمد على:
  - `useProducts` من `src/app/context/ProductContext.tsx`
  - `ProductCard` من `src/app/components/ProductCard.tsx`
- المصدر البياني:
  - `ProductContext` -> `fetchAllProducts` في `src/lib/services/productsService.ts`
  - Supabase table: `products`

### `/products`
- Page: `src/app/pages/Products.tsx`
- يعتمد على:
  - `useProducts`
  - `ProductCard`
- المصدر البياني:
  - نفس مسار `/`

### `/products/:id`
- Page: `src/app/pages/ProductDetail.tsx`
- يعتمد على:
  - `useProducts`
  - `useCart`
- المصدر البياني:
  - قراءة من `ProductContext`
  - إضافة للسلة عبر `CartContext` (state + local storage)

### `/about`
- Page: `src/app/pages/About.tsx`
- لا يعتمد على خدمة بيانات مباشرة.

### `/contact`
- Page: `src/app/pages/Contact.tsx`
- يعتمد على:
  - `submitContactForm` من `src/lib/services/contact.ts`
- ملاحظة:
  - الخدمة الحالية Placeholder، ليست مسار backend فعلي.

### `/login`
- Page: `src/app/pages/Login.tsx`
- يعتمد على:
  - `useAuth` من `src/app/context/AuthContext.tsx`
- المصدر البياني:
  - `supabase.auth.signInWithPassword(...)`
  - تحميل الدور من `user_roles`

### `/dashboard`
- Gate: `src/app/components/ProtectedRoute.tsx`
  - يتطلب تسجيل دخول + `isAdmin`.
- Page: `src/app/pages/Dashboard.tsx`
- Tabs (lazy loaded):
  - `DashboardOverview` -> `fetchDashboardOverviewStats` (`ordersService`)
  - `DashboardProducts` -> CRUD (`productsService`)
  - `DashboardOrders` -> `fetchOrders`, `deleteOrderAndItems` (`ordersService`)
  - `DashboardCustomers` -> `fetchCustomerStats` (`customersService`)
  - `DashboardSettings` -> `fetchStoreSettings`, `updateDeliveryPrice` (`storeSettingsService`)

## 3) مسار Checkout الآمن

- UI trigger: `Header` -> `CheckoutDialog`
- Submit path:
  - `CheckoutDialog` -> `submitCheckoutOrder` في `src/lib/services/checkoutService.ts`
  - `checkoutService` -> `supabase.rpc("create_order_secure", ...)`
- SQL authority:
  - `db/migrations/20260321_phase1_security_up.sql`
  - الوظيفة: `public.create_order_secure`
- الدالة تنفذ:
  - التحقق من البيانات
  - تسعير من `products`
  - قراءة `store_settings.delivery_price`
  - إنشاء `customers/orders/order_items`
  - خصم المخزون

## 4) الارتباطات الأمنية (Auth/RLS)

- مصادقة: Supabase Auth session.
- صلاحيات Admin:
  - تحميل الدور من جدول `user_roles`.
- تفويض بيانات:
  - RLS policies على `products/customers/orders/order_items/store_settings/user_roles`.
- مراجع:
  - `db/migrations/20260321_phase1_security_up.sql`
  - `docs/architecture-security-matrix.md`

## 5) نقاط ازدواجية/تشعب مسار (يجب ضبطها)

- وجود أكثر من ملف SQL يغطي نفس النطاق:
  - `database_schema.sql`
  - `secure_checkout_rpc.sql`
  - `db/migrations/*` (المصدر الصحيح المقترح).
- وجود API قديم `server/index.ts` لا يمثل المسار الأساسي للتطبيق الحالي.
