# تقارير المشروع

هذا المجلد هو المرجع الرسمي لتحليل الارتباطات، جودة الهيكلية، وأفضل مسار تطوير آمن وقابل للتوسع.

## الهيكل

1. `الخصائص/تطويرات`  
   يحتوي على خريطة الارتباطات الكاملة + خارطة تطوير تنفيذية.

2. `الهيكلية-الصحيحة-لإضافة-الخصائص`  
   يحتوي معيار رسمي لإضافة أي خاصية جديدة بشكل منظم وآمن.

3. `خصائص-لا-تتبع-أفضل-الممارسات`  
   يحتوي تشخيصًا دقيقًا لمواضع عدم التنظيم، التكرار، واختلاط المسؤوليات.

## طريقة الاستخدام

- قبل أي تطوير جديد: اقرأ `الهيكلية-الصحيحة-لإضافة-الخصائص/معيار-إضافة-خاصية-جديدة.md`.
- قبل أي refactor: راجع `خصائص-لا-تتبع-أفضل-الممارسات/تشخيص-الخصائص-غير-المنظمة.md`.
- عند التخطيط للإصدارات: استخدم `الخصائص/تطويرات/خطة-تطوير-مرجعية.md`.
